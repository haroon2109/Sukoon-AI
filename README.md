# Sukoon AI - Enterprise Trust & Safety Infrastructure

Sukoon AI is an API-first platform that automatically intercepts, verifies, and neutralizes viral misinformation, deepfakes, and hate speech. We provide the "Stripe for Misinformation"—equipping social platforms (Meta, X, WhatsApp) and massive Newsrooms with a localized, legally defensible RAG verification engine.

## 🚀 The Moat
Unlike generic LLMs that hallucinate, Sukoon AI is anchored by a specialized RAG (Retrieval-Augmented Generation) pipeline directly indexing authoritative Indian fact-checkers:
- **PIB Fact Check** (Government policy & economic claims)
- **Alt News** (Communal & geopolitical misinformation)
- **BOOM Live** (Deepfakes and multimedia forensics)

To minimize AI hallucinations and handle viral fake news efficiently, Sukoon AI implements an internal knowledge ring using a hosted Qdrant Vector database on Google Cloud. Instead of blindly sending every request to a massive LLM, our backend performs a semantic vector search against locally verified facts first. If a piece of misinformation has already been debunked by community admins, Sukoon AI retrieves it instantly in milliseconds, saving massive operational token costs and preventing the system from generating contradictory advice.

### ⚖️ LLM Comparator (Hallucination Reduction)
During our development and testing phases, we rigorously battle-tested our fact-checking models using **Google AI's open-source LLM Comparator**. By visually analyzing and comparing how different prompt variations and constraints affected our truth engine's reasoning logic side-by-side, we systematically eliminated edge cases and reduced model hallucinations. This ensures that every Explainable AI report generated by Sukoon AI is perfectly grounded in retrieved evidence rather than generative guesswork.

### 🧠 Explainable AI (XAI) Architecture
Explainable AI means the system doesn't just output a "False" verdict; it transparently tells the user exactly *why* it reached that conclusion. 

| XAI Feature | What It Displays to the User | Free Tool Powering It |
| :--- | :--- | :--- |
| **Toxicity Scan** | Confirms the incoming claim doesn't violate safety guardrails. | **Perspective API** / **ShieldGemma** |
| **Fact Grounding** | Highlights exactly which trusted database entries contradicted the claim. | **Qdrant** (Vector Search) |
| **Confidence Gauge** | A visual percentage indicating how certain the model is based on cross-referencing. | **Gemini** (via JSON schema) |
| **Verification Trace** | A structured, natural language breakdown of the AI's logical reasoning steps. | **Gemini** (via Structured Outputs) |

### 📈 Langfuse Telemetry & Observability
Sukoon AI integrates **Langfuse**, an open-source LLM engineering and observability platform. Every time a user submits a claim for verification, Langfuse maps out the entire multi-step chain (Toxicity Check → Claim Extraction → Qdrant Vector Search → Gemini Evaluation). This enterprise-grade telemetry tracks:
- **Execution Traces**: Visualizes the exact conversational paths and function calls.
- **Cost Analytics**: Tracks precise token usage across all steps.
- **Latency Monitoring**: Identifies exactly which agent or database query is causing bottlenecks.
If an AI verification goes wrong or hallucinates, platform administrators can instantly pull up the Langfuse trace ID to pinpoint exactly which node in the agentic workflow failed.

### 🛡️ Prism & Agent Memory Guard (Responsible AI)
To ensure adversarial attackers cannot hijack the truth engine, Sukoon AI is hardened against prompt injections and memory-poisoning using **Prism & Agent Memory Guard**. This open-source runtime evaluation and security layer acts as an autonomous firewall, continuously monitoring agent logic and incoming claims to guarantee that malicious payloads disguised as fake news cannot break our prompt guardrails.

## 🏗️ Architecture & Orchestration

### 🌟 The Vision: An Agentic Verification Loop
Sukoon AI maps open-source workflow frameworks (LangGraph, Google ADK) into a highly cohesive multi-agent system:
1. **Task Ingestion & Delegation:** Triggered via API. A user flags a suspicious URL. The orchestration framework intercepts the request and delegates the root task to a supervisor agent.
2. **Multi-Agent Tool Calling:** Parallel Execution. The supervisor calls **Agent A (Scraper)** to pull text, while concurrently calling **Agent B (Database Specialist)** to look for cross-references inside our Qdrant vector embeddings.
3. **Synthesis and Debate:** Intelligent Reasoning. Both agents return their results to **Agent C (The Truth Evaluator)**. Using Gemini, this agent synthesizes the findings and drafts a structured JSON verification report specifying confidence scores and logical fallacies.
4. **Downstream Workflow Trigger:** Automated Actions. The framework finishes the workflow by securely triggering downstream webhooks (e.g., our self-hosted **n8n** automation nodes) to notify community moderators or broadcast a retraction.

### 🤖 Core Verification (Gemini 2.5 Flash)
The Truth Engine completely offloads multimodal processing to Google AI Studio's Developer API. By configuring **Gemini 2.5 Flash** with a strict `response_schema` (via Pydantic), Sukoon AI guarantees precisely structured JSON outputs (verdict, confidence score, citations) perfectly formatted for the frontend dashboard—while operating safely within the permanent free tier limit of 1 million tokens.

### ⚡ Local Celery Task Queue (SQLite)
To keep the infrastructure incredibly lightweight and cost-effective on Cloud Run, Sukoon AI uses **Celery with a localized SQLite database broker**. Instead of paying for an expensive managed Redis instance, this open-source architecture seamlessly queues and processes heavy background tasks directly within the container during periods of light traffic.

### 🕵️ Playwright & Beautiful Soup (Hybrid Scraper)
To accurately verify links, Sukoon AI employs an advanced open-source scraping stack:
- **Tier 1 (Dynamic)**: Uses **Playwright** (headless Chromium) to fully render JavaScript-heavy public platforms and social media sites.
- **Tier 2 (Static Fallback)**: If Playwright fails or times out, the system seamlessly falls back to lightning-fast **Requests + Beautiful Soup** parsing.
This completely eliminates the need for expensive, commercial web scraping APIs while maintaining enterprise-grade extraction capabilities.

### 🧠 Free Tier Google AI Embeddings
Rather than relying on expensive, commercial Vertex AI pipelines, Sukoon AI utilizes Google's native **`text-embedding-004`** model via the `google.generativeai` SDK. This allows the RAG (Retrieval-Augmented Generation) pipeline to generate state-of-the-art vector dimensions absolutely free of charge using the AI Studio Free Tier API Key.

### 🗄️ Supabase & pgvector (Open-Source RAG)
To prove the enterprise scalability of the Truth Engine, the RAG memory bank operates on **Supabase** (open-source PostgreSQL) using the native **pgvector** extension. This allows Sukoon AI to store generated semantic embeddings directly alongside factual text records and perform real cosine-similarity math (`<=>`), bypassing the need for expensive specialized vector databases like Pinecone.

Sukoon AI handles immense scale using an asynchronous fan-out/fan-in processing pipeline.

### 🔌 Google ADK (Agent Development Kit) & MCP
To maintain an extensible and standardized agentic architecture, Sukoon AI is built directly on **Google's Agent Development Kit (ADK)**—an official open-source (Apache 2.0) Python framework designed specifically for runtime execution of agents. Using ADK directly proves that Sukoon AI is building a **Google-aligned ecosystem**. It provides opinionated structures for tool calling, step-by-step agent loops, and built-in local debugging interfaces. By wrapping internal scripts as **Model Context Protocol (MCP)** tools inside the ADK, our Gemini agents can natively and securely interface with backend datasets using a universal protocol.

### 🧠 LangGraph (Stateful Agent Orchestration)
To empower platform administrators and journalists with deep investigative capabilities, Sukoon AI orchestrates its multi-turn conversational agents using **LangGraph**. Instead of a static "True/False" output, LangGraph manages the complex, stateful conversation memory. If an analyst asks, "Why did you flag this image as altered?" or "Can you show me the original source article?", the LangGraph orchestrator seamlessly routes the request to the correct forensic agent while maintaining the context of the entire investigation.

## 💻 Tech Stack
- **Frontend**: Next.js 15 (App Router), React, Tailwind CSS, ShadCN, Framer Motion
- **Backend API**: FastAPI, WebSockets
- **Async Processing**: Celery, Redis
- **Database / Cache**: PostgreSQL, Qdrant (Vector DB for RAG edge caching)

## 🎨 Design Philosophy: "Clinical & Calm"
Sukoon AI deals with high-anxiety viral content. To instantly de-escalate emotional states, the UI entirely avoids bright "startup" colors. 
- **Colors**: Muted earth tones (`stone-50`), deep navy (`slate-900`), and clinical `teal-700`.
- **Typography**: Tracked-out uppercase micro-copy to establish a forensic, structured layout.
- **Components**: Pillowy cards, extremely soft ambient shadows, and slow-pulsing animations.

## 🛠 Running the MVP Locally

### 1. Start the FastAPI Backend
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### 2. Start the Next.js Frontend
```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:3000` to view the Developer API Landing Page, or navigate to `http://localhost:3000/dashboard` to access the Analyst Command Center.
